# Method Design

Use this skill once the problem framing is stable enough to support a method argument.

## Main job

Turn a positioned research idea into a narrow, defensible method section.

## Workflow

1. Restate the paper's problem, unit of analysis, and intended outcome.
2. Name the method type:
   - conceptual framework
   - system design
   - pipeline
   - empirical method
   - mixed design
3. Define the main components or constructs.
4. If relevant, separate clearly:
   - mathematical model
   - simulation environment
   - physical apparatus or platform
   - sensing and data acquisition
   - control or optimization logic
5. Explain the logic linking components to the problem.
6. Add decision points, assumptions, and constraints.
7. Make the section legible for later evaluation by noting what could be measured or compared.

## Output shape

Prefer:

- section outline
- subsection headings
- concise prose draft
- assumptions and limitations note

## Guardrails

- Do not let the method read like a product brochure.
- State units, variables, conditions, and assumptions where they matter.
- Keep component descriptions tied to the research question.
- If there is no real empirical plan yet, say so and keep the method conceptual.
- Build in traceability so later evaluation criteria make sense.
