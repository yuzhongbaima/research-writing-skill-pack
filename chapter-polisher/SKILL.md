# Chapter Polisher

Use this skill late in the workflow, after the main argument of the section is already decided.

## Workflow

1. Identify the section's job in the paper.
2. Check internal structure:
   - opening purpose
   - logical paragraph order
   - consistent terminology
   - explicit transitions
3. Flag unresolved problems:
   - unsupported claim
   - placeholder argument
   - missing citation
   - missing unit, condition, or boundary specification
   - abrupt leap in reasoning
4. Revise for:
   - precision
   - shorter sentences where useful
   - academic tone without unnecessary inflation
   - Markdown cleanliness for export

## Output shape

- revised section text
- short issue list for anything still unresolved
- optional export notes if the section will be turned into PDF or TeX next

## Guardrails

- Do not silently repair substantive evidence gaps with style edits.
- Preserve the author's voice where possible.
- Prefer small structural fixes over stylistic churn.
