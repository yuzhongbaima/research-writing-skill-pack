# Chinese Prompt Templates

This file provides copy-ready Chinese prompts for the research-writing suite.

Before using a template, replace placeholders such as:

- `[研究主题]`
- `[研究领域]`
- `[研究对象/系统]`
- `[目标期刊/会议]`
- `[已有材料]`
- `[我最担心的问题]`

## 0. Router Prompt

适合先让套件判断当前该走哪一步。

```text
我正在做一篇[研究领域]论文，主题是[研究主题]，研究对象/系统是[研究对象/系统]。
我当前已有的材料包括：[已有材料]。
我的目标是投稿到[目标期刊/会议]，如果目前还不适合投稿，也请直说。

请先不要直接大段改写正文，而是先判断我现在最该进入哪一个工作阶段：
1. 科研监督与流程门禁
2. 中途诊断与共同作者式讨论
3. 文献综述
4. 研究定位
5. 方法设计
6. 图表与结果叙事
7. 引用审计
8. 审稿人预审
9. 章节润色

请输出：
1. 建议我当前进入的阶段
2. 为什么是这一步而不是下一步
3. 当前最值得先产出的 1 个中间产物
4. 如果继续推进，后续 2 到 3 步应该怎么接

如果我现在给你的不是完整开题材料，而是半成品想法、局部手稿、结果图表或审稿意见，也请先判断我当前所处状态，再决定如何介入，不要默认让我从头重来。
```

## 1. Research Supervision

```text
我希望你帮助我搭一套“半自动科研工作流”，但不要把它理解成可以全自动产出高水平论文。

研究领域：[研究领域]
研究主题：[研究主题]
研究对象/系统：[研究对象/系统]
目标期刊/会议：[目标期刊/会议]
我已有的自动化能力：[例如文献检索、代码执行、画图、草稿写作]
我明确要求必须人工把关的环节：[例如文献筛选、实验设计、结果解读、最终 claim]

请你作为科研监督者，帮我输出：
1. 一套适合该课题的阶段化流程
2. 哪些步骤可以自动化，哪些步骤必须人工审批
3. 每一步应沉淀哪些中间产物
4. 哪些“强制约束”值得加进去，避免论文看起来完整但科学上不可靠
5. 如果目标瞄准 Nature Communications 水平，当前流程最不能妥协的 3 到 5 条规则

要求：
- 不要默认“一个想法直接生成一篇论文”
- 不允许把模拟结果或 AI 编造的数字混进投稿稿
- 自动化的目标是提速和增广，不是替代科学判断
```

## 2. Midstream Diagnosis and Coauthoring

```text
我现在不给你完整开题材料，而是给你一个中间状态的半成品。请你不要只做表面润色，而是像共同作者一样帮我诊断、重构和推进。

研究领域：[研究领域]
研究主题：[研究主题]
研究对象/系统：[研究对象/系统]
目标期刊/会议：[目标期刊/会议]
我现在给你的材料类型：[半成品想法/方法草图/图表/局部手稿/完整手稿/审稿意见]
材料内容：[已有材料]
我目前最担心的问题：[我最担心的问题]

请输出：
1. 先判断我现在处于论文推进的什么状态
2. 哪些部分已经相对成立
3. 哪些部分只是直觉、需要补证据或重构
4. 当前最危险的 3 个研究或写作风险
5. 你建议我下一步最值得做的 1 到 2 个动作
6. 如果你认为我的 framing、方法或 claim 有更优版本，请直接提出替代方案

要求：
- 不要默认让我从头开始
- 不要只顺着我现有写法润色
- 要明确区分“已有事实”“基于材料的推断”“尚待验证的假设”
- 可以适当使用更强的分析和推理，帮助我完善课题、研究、方法和论文
```

## 3. Literature Review

```text
请作为[研究领域]方向的科研助理，帮我做一轮面向论文写作的文献综述准备。

研究主题：[研究主题]
研究对象/系统：[研究对象/系统]
目标问题：[我要解决的具体工程问题]
预期投稿方向：[目标期刊/会议]
我已有的文献或关键词：[已有材料]

请按下面要求输出：
1. 先给出这类问题应重点关注的 4 到 6 个文献主题簇
2. 对每个主题簇说明：
   - 主要研究问题
   - 常见方法路线
   - 常见验证方式（仿真/实验/半实物/现场）
   - 目前常见不足
3. 帮我列一个适合后续写 Related Work 的综述矩阵字段
4. 单独指出哪些地方最可能形成我的研究空白

注意：
- 不要只堆论文摘要
- 要特别关注工况、边界条件、验证强度和应用场景差异
- 如果某类证据主要来自仿真，请明确标出来
```

## 4. Research Positioning

```text
请根据我目前的研究材料，帮我把论文定位收窄成一个更适合工科论文的研究命题。

研究主题：[研究主题]
研究领域：[研究领域]
研究对象/系统：[研究对象/系统]
已有发现/初步想法：[已有材料]
我想体现的价值：[例如安全、能耗、排放、效率、鲁棒性、部署可行性]

请输出：
1. 用通俗但准确的话重述我的论文到底要解决什么问题
2. 给出 2 到 4 个可选的研究定位版本
3. 对每个版本说明：
   - 核心 gap
   - 贡献边界
   - 可能被审稿人质疑的点
4. 选出一个你最推荐的版本，并给出：
   - 一句话 problem statement
   - 一句话 gap statement
   - 2 到 3 条 contribution 表述

要求：
- 不要强行套“新算法超过 SOTA”的叙事
- 如果更适合写成工程系统、验证框架、控制策略或部署研究，请明确指出
- 控制 claim 强度，避免过度包装
```

## 5. Method Design

```text
请帮我把下面这个研究定位，扩展成一版适合工科论文的 Method/Section 3 结构。

研究定位：[贴上 problem statement / gap / contribution]
研究对象/系统：[研究对象/系统]
已有方法构想：[已有材料]
我手里真实可做的验证条件：[仿真/实验平台/现场数据/飞行测试/水池试验等]

请输出：
1. 一个 Section 3 的分节大纲
2. 每一小节应该回答什么问题
3. 哪些地方必须写清楚变量、单位、边界条件、约束和假设
4. 哪些内容应该归入：
   - 数学模型
   - 仿真设置
   - 实验装置/平台
   - 感知与数据采集
   - 控制/优化逻辑
5. 给一版 500 到 800 字的中文方法草稿

要求：
- 不要写成产品说明书
- 要和研究问题强绑定
- 如果我现在证据还不足以支持“完整实验方法”，请帮我改成更诚实的概念性或半实证写法
```

## 6. Figure Table Storytelling

```text
请帮我把下面的结果/方法内容，设计成适合论文投稿的图表叙事方案。

研究领域：[研究领域]
研究主题：[研究主题]
我要表达的核心结论：[一句话]
现有结果或素材：[已有材料]
目标期刊/会议：[目标期刊/会议]

请输出：
1. 最适合做成图的内容有哪些
2. 最适合做成表的内容有哪些
3. 对每个图/表给出：
   - 图表类型建议
   - 标题建议
   - caption 草稿
   - 应标注的单位、工况、变量、误差线或注释
4. 如果我现在的图表思路有误，请直接指出并改成更合理的方案

要求：
- 不要默认所有比较都用柱状图
- 如果存在连续变化、敏感性、Pareto 权衡或不确定性，应优先考虑更合适的图型
- caption 不要只是改写标题，要说出“读者应该看到什么”
```

## 7. Citation Audit

```text
请对下面这段论文草稿做一轮引用审计，重点不是润色，而是检查每个重要 claim 是否真的被文献支撑。

研究领域：[研究领域]
目标期刊/会议：[目标期刊/会议]
草稿内容：[贴正文]
参考文献线索：[已有文献/引用列表]

请输出一个表格，列出：
1. 关键 claim
2. 当前对应引用
3. 支撑强度（直接支撑/部分支撑/弱代理支撑/无支撑）
4. 问题类型
5. 最保守可行的修复建议

额外要求：
- 区分同行评审论文、标准规范、技术报告、手册类材料
- 不要虚构缺失引用
- 如果一句话的力度超过了现有证据，请优先建议降调，而不是强行补 citation
```

## 8. Reviewer Preflight

```text
请把下面这篇论文草稿当作[目标期刊/会议]的投稿候选稿，从审稿人的角度做一轮预审。

研究领域：[研究领域]
目标期刊/会议：[目标期刊/会议]
论文摘要/正文：[已有材料]
我最担心的问题：[我最担心的问题]

请输出：
1. 你认为最可能影响录用/接收的前 5 个问题
2. 每个问题的严重程度
3. 每个问题为什么会被 reviewer 卡住
4. 每个问题最实际的修复路径
5. 如果不补实验，只能改 framing，哪些问题可以通过收窄 claim 来缓解

要求：
- 区分“方法缺陷”“验证不足”“表述不清”三类问题
- 不要只给泛泛建议
- 如果我是工科论文，请特别检查工况、边界条件、验证层级、工程可行性、安全/法规相关风险
```

## 9. Chapter Polisher

```text
请帮我润色下面这一节论文，但重点是学术表达的清晰度和论证完整性，而不是空泛地“写得更高级”。

研究领域：[研究领域]
本节功能：[例如 Introduction / Related Work / Method / Results / Discussion]
目标期刊/会议：[目标期刊/会议]
待修改内容：[贴正文]

请输出：
1. 先指出这节最影响可读性的 3 到 5 个问题
2. 再给出修改后的完整版本
3. 最后列出仍然没法通过纯润色解决的问题，例如缺 citation、缺工况说明、逻辑跳步等

要求：
- 保留原意，不要擅自增加没有证据的 claim
- 避免明显 AI 味表达、空泛抬升、模板化过渡句
- 工科内容里不要丢掉单位、变量、边界条件和条件限定语
```

## 10. Transportation Add-on

```text
这是交通运输工程方向，请在分析时额外关注：
- 网络尺度、交通方式、时间粒度、预测/控制时域
- 数据来源是真实运行数据、仿真数据还是混合
- 指标是否覆盖延误、通行效率、安全、排放、鲁棒性
- 是否把仿真结果过度外推到真实部署
```

## 11. Energy and Environment Add-on

```text
这是节能环保方向，请在分析时额外关注：
- 系统边界、功能单位、能量/物料衡算
- 碳排、污染物、去除率、能耗、成本指标是否定义清楚
- 结论是否区分实验室尺度、试点尺度和工程应用尺度
- 经济性和环境收益是否有足够证据支撑
```

## 12. Ocean Engineering Add-on

```text
这是海洋工程方向，请在分析时额外关注：
- 海况、水深、载荷、耦合效应、尺度效应
- 证据来自 CFD、水池试验、海试还是理论分析
- 结论是否超出了测试海况或运行包线
- 稳定性、疲劳、控制性能和海上部署约束是否说清楚
```

## 13. Low-Altitude Aircraft Add-on

```text
这是低空航空器方向，请在分析时额外关注：
- 飞行包线、任务剖面、载荷、航时、气象和通信约束
- 证据来自仿真、半实物、飞行试验还是运行数据
- 安全、监管、失效模式和实际部署条件是否交代充分
- 是否把仿真或小规模验证过度外推为真实运行能力
```
