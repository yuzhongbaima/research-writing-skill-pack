# Workflow Map

This suite follows an eight-stage paper workflow.

The stage order is canonical, but entry into the workflow is nonlinear.

## Entry logic

Users may enter from any stage with partial materials. The first task is to diagnose where they really are and what the smallest high-leverage next move should be.

Do not reset the project to stage 1 by reflex.

## 1. Research Supervision

Goal: set automation scope, human gates, and artifact discipline before the project accelerates.

Outputs:

- supervision plan
- approval gates
- artifact manifest
- claim discipline rules

## 2. Literature Review

Goal: build an evidence base before making claims.

Outputs:

- paper and source list with relevance notes
- theme clusters
- tensions, limitations, and open questions

## 3. Research Positioning

Goal: turn a broad topic into a narrower, arguable paper stance.

Outputs:

- problem framing
- gap statement
- thesis variants
- tentative contributions

## 4. Method Design

Goal: convert the positioned argument into a clear method section.

Outputs:

- method skeleton
- construct definitions
- apparatus, simulation, or system boundary notes
- evaluation hooks

## 5. Figure and Table Storytelling

Goal: turn architecture, process, or results into visual evidence and captions that actually help the paper.

Outputs:

- plot recommendations
- caption drafts
- table structures
- axis, unit, and annotation checks

## 6. Citation Audit

Goal: verify that each important claim has the right citation support.

Outputs:

- unsupported-claim list
- weak-citation list
- bibliography cleanup notes

## 7. Reviewer Preflight

Goal: inspect the draft from the perspective of a skeptical reviewer and catch nontrivial risks before submission.

Outputs:

- top-priority reviewer concerns
- venue-fit or journal-fit checklist
- missing validation or framing notes

## 8. Chapter Polishing

Goal: make one section internally consistent and export-ready.

Outputs:

- revised section draft
- list of unresolved gaps
- export notes

## Decision logic

- Diagnose before routing when the user provides mixed or half-finished materials.
- Do not draft a strong method section until the positioning is stable enough to answer: what exact problem, for whom, under what assumptions, and why existing work is insufficient.
- Set gates before acceleration. Speed is useful only if rollback is still possible.
- Do not design visuals as decoration. Every figure and table should carry argumentative load.
- Do not polish heavily before running at least a lightweight citation audit.
- Run reviewer preflight before final polishing if the draft is close to submission.
- For ambitious journal targets, never skip human review of literature screening, experiment design, or final claim strength.
- If the draft structure is wrong but the core evidence is sound, fix the structure instead of restarting the project.
- If the user is blocked, produce the smallest next artifact that reduces ambiguity.
